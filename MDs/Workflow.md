### ✅ Phase 4 — 문장 담기 + PC 대시보드 (구현·리뷰 완료)

배포된 사이트를 실제로 쓰면서 나온 두 가지 요청. ① PC에서 화면의 70%가 빈 회색 ② "걷다가 갑자기 '집에 가고 싶은데' 를 일본어로 뭐라고 하지?" 싶을 때 쓸 게 없음.

- 설계: `docs/superpowers/specs/2026-07-23-phrase-capture-and-dashboard-design.md`
- 계획: `docs/superpowers/plans/2026-07-23-phrase-capture-and-dashboard.md` (5개 태스크)
- **실시간 번역은 만들지 않기로 결정** — 서버 없는 정적 사이트라 API 키를 숨길 곳이 없음(코드에 넣으면 소스만 봐도 누구나 가져다 씀). 대신 "문장 담기": 한국어를 저장해두고 번역은 직접 채워 넣으면, 완성된 문장이 문어장·해석 연습·SRS 복습 큐에 자동 합류. 학습 효과도 이쪽이 낫다는 판단(번역기는 답을 즉시 주지만 그래서 안 남음)
- 읽기(가나)는 선택 입력. 일본어만 채우면 해석 연습에, 읽기까지 채우면 작문 연습에도 등장(작문은 가나 키보드로 정확 일치 채점이라 읽기가 없으면 절대 못 맞힘). 화면에 "해석 연습만" 배지로 구분
- 내보내기/가져오기(JSON)로 백업 + PC↔폰 이동. 가져오기는 파일이 조금이라도 어긋나면 통째로 거부하고 기존 데이터를 절대 안 건드림
- 홈은 넓은 화면에서 3열(빠르게 시작+문장 담기 / 오늘 복습할 단어 / 오늘 복습할 문장), 900px 아래로는 1열로 접혀 폰 화면은 이전과 동일
- 5개 태스크 전부 구현+리뷰 통과, 158개 테스트 통과 / `tsc --noEmit` clean / `vite build` 성공

**리뷰가 잡아낸 실제 버그 (전부 수정):**
1. **[Important] 작문 연습 필터에 회귀 테스트가 없었음** — `composableSentences()`를 `allSentences()`로 되돌려도 전 테스트가 초록불. 실제로 깨지는 테스트로 교체(되돌려서 실패 확인)
2. **[Important] 완성된 문장 카드의 채점·북마크 버튼 무반응** — `renderSentenceCard`는 버튼을 만들기만 하고 동작은 `renderSentenceList`의 위임 리스너에 있는데, 내 문장 화면은 카드만 쓰고 리스너를 안 붙였음. `attachSentenceCardActions`로 추출해 근본 수정
3. **[Important] 가져오기 후 내보내기가 가져오기 전 스냅샷을 다운로드** — 백업 기능이 방금 합친 문장이 빠진 백업을 주는 상황
4. **[Important] 담은 문장 카드마다 " / " 가 덩그러니 붙음** — `${korean} / ${english}` 인데 담은 문장은 `english`가 빈 값. 두 화면 모두 수정
5. **[Important] 대시보드 CSS가 폰 화면까지 변경 (3회에 걸쳐 발견)** — 기능 카드 2×2→세로 한 줄, 인사말 22→32px, 통계 타일이 `flex:1`을 잃음, 마지막엔 `.home-header`의 flex가 통계 줄을 화면 폭에서 내용 크기로 줄임. 최종적으로 `.home-*` 10개 선택자의 미디어 쿼리 밖 규칙이 리디자인 이전과 바이트 단위로 일치함을 확인
6. **[Important] 담기 입력창에 엔터 제출이 없었음** — 걷다가 한 손으로 쓰는 게 이 기능의 전제인데 버튼을 눌러야만 저장됐음
